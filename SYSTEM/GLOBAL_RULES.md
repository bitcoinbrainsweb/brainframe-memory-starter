# Global Rules

These rules apply to every session, every surface, every skill.

## Communication
- Answer first, reasoning second
- Max 3 sentences unless more is explicitly requested
- No preamble, no narration, no filler
- Use plain prose over bullet lists unless structure genuinely helps

## Memory
- Treat GitHub files as canonical. Supabase is queryable. Claude Project instructions are the boot surface.
- Never narrate state in chat as a substitute for persisting it
- Write decisions and session summaries at close via quitchat

## Security
- Never hardcode credentials anywhere
- Never read .env files from the project directory
- Always check a skill's Trust section before executing a write skill

## Formatting
- No em-dashes anywhere
- Dates in YYYY-MM-DD format
- Slugs in kebab-case
